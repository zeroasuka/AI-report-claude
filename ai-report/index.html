CH 03 · 自動化進階
CHAPTER 03 · 進階加掛

Agent 不只是「下指令」，
它還能 自己跑、自己學、自己排班。

SKILL · 教 AI 專業

🧠 Skill

給 AI 一個「專業領域包」——它就學會該領域的規矩、慣例、API。

📦 範例：

html-ppt deep-research verify

這份簡報就是用 html-ppt skill 做的——一行指令就裝好版型 + 動畫。

HOOK · 條件觸發

🪝 Hook

「每當 X 發生時，幫我 Y」—— 不用主動呼叫，AI 自己接管。

📦 範例：

  • 每次存檔 → 自動跑 lint
  • 每次 commit → 自動寫 commit message
  • 收到 Slack → 自動分類進 inbox
CRON · 自動排班

⏰ Scheduled Task

「每天 9 點幫我做 X」—— AI 真的會在那個時間主動做事。

📦 範例：

  • 每早 8:50 整理 inbox
  • 每週五匯出工時報告
  • 每小時檢查伺服器健康

🚀 這三件事疊加起來，
Agent 從「被動的工具」變成「主動的同事」。

⏱ 1m30s · Ch3 自動化加掛（新概念） • Agent 之上還能加三件事讓它升級 • Skill：給 AI 一個專業領域包（這份簡報就是用 html-ppt skill 做的） • Hook：條件觸發（每次存檔 → 自動跑 lint） • Scheduled：排班（每天 8:50 整理收件匣） • 收：Agent 從「被動工具」變「主動同事」— 2026 年最大差別 → Ch3 一句話收尾

CH 03.5 · Model vs Agent
CORE · Model vs Agent

Model 是 Model，Agent 是 Agent

你不是在跟一個聊天機器人講話，而是驅動一個 Agent。

CENTER · 腦

Model

GPT-5 Claude Opus Gemini DeepSeek Ollama Qwen 3.6 MiniMax M3 GLM-5

🧠Model（模型）

只活在對話框裡
  • 產生文字、推理、答題
  • 不會動手，只能講話
  • 換模型 ＝ 換腦
  • 例：GPT-5、Claude、Gemini

🤖Agent

模型 ＋ 動手能力
  • 讀檔、改檔、跑指令、調工具
  • 有權限邊界、有交付產物
  • 換工具 ＝ 換包裝，配料盒一樣
  • 例：Codex、Claude Code、OpenClaw
理解這層差異，後面所有名詞才不會混淆。
⏱ 1.5m · 主軸建立 • 主視覺：Model 在中央，8 個工具圍著它＝Agent • 用對照表把 Model 與 Agent 講清楚（換模型 vs 換工具） → P13 4 種介面

CH 03.5 · 配料盒
INGREDIENTS · 6 種配料

Skill、MCP、AGENTS.md…到底是什麼？

這些名詞不是並列功能，而是 Agent 圍繞 Model 的 6 種配料。

🧩
Skill 專業領域包
DOMAIN PACKAGE

把規矩、範例、API 慣例包成一個資料夾，Agent 遇到相關情境就會自動照做。

💡 用法：寫一份 Code Review Skill，下次開 PR 就自動照規範檢查。
🔌
MCP Model Context Protocol
EXTERNAL SOCKET

接外部工具的插座規格。Slack、Notion、Drive、自家 DB 都能接上；4 種介面共用同一份設定。

💡 用法：裝 Slack MCP，Codex 能讀你的訊息、發通知。
📄
Markdown 文件
SPEC / NOTES

給 Agent 看的說明書。需求、規範、TODO 寫成 .md，Agent 自己讀。

💡 用法：寫 SPEC.md 描功能、STYLE.md 描規範，動手前 Codex 會掃過。
🗃️
資料庫 Database
LONG-TERM MEMORY

給 Agent 的長期記憶。SQL / 向量 DB / 知識庫，比塞進對話框更可靠。

💡 用法：把過去 PR、issue embed 進向量 DB，Codex 用 MCP 查。
🤖
Agent 智能體
THE WHOLE THING

能動手的智能體本身。會讀檔、改檔、跑指令、調工具、產出可驗收的東西。

💡 用法：Codex 本身就是一個 Agent；Claude Code、OpenClaw、Hermes 也是。
📋
AGENTS.md 專案規範
PROJECT RULES

放在專案根目錄，告訴 Agent 這個 repo 的風格、慣例、不能碰的檔。

💡 用法：寫好它，4 種介面進這個專案都會自動讀，行為一致。
名詞會一直冒新的，但位置就那幾個——知道每個放在哪，就不會迷路。
⏱ 2m · 名詞解釋 • 6 個常被混淆的名詞：Skill、MCP、Markdown、DB、Agent、AGENTS.md • 重點：工具換 = 包裝換，配料盒還是這些 → P15 五層工作流

CHAPTER 05 · 4 個動作

它不是回答問題，
它是動手解決問題。

STEP · 01

🔍 診斷

看截圖 → 判斷是 MSVCR100.dll 缺失，
對應 Visual C++ 2010 runtime。

STEP · 02

🔬 釐清

System32 有 64-bit、SysWOW64 沒有，
所以缺的是 32-bit 版。

STEP · 03

🔐 驗證

從 MS 官方下載 vcredist_x86.exe，
先驗數位簽章 = Microsoft。

STEP · 04

✅ 執行

安裝 → 確認 DLL 已存在 → 回報，
整段 ExitCode = 0。

💡 以前 AI 是「**告訴你怎麼修**」，
現在 AI 是「**它幫你修好了**」。
